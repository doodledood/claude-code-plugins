# Definition: Improve writing skill with patterns from Wikipedia:Signs of AI writing

## 1. Intent & Context

- **Goal:** Fold Wikipedia:Signs of AI writing patterns into the existing writing plugin so the human-writing skill, its research reference, and the writing-reviewer agent detect a wider, more current set of AI tells. Restructure the existing reference thematically; surface top-tier additions to the SKILL.md; extend the reviewer with frequency-aware judgment.
- **Mental Model:**
    - The skill's primary consumer is an LLM, not a human reader — content optimizes for LLM-readable structure (do/don't tables, scannable rules), not prose explanation.
    - Wikipedia article is acquired locally at `/tmp/signs-of-ai-writing.md` (saved by the user before /do runs); not archived in the repo.
    - Distinct additions vs current skill: era-tracked vocabulary (GPT-4 era 2023–mid-2024 vs GPT-4o era mid-2024–mid-2025), heading capitalization (title case all main words), curly quotes (ChatGPT/DeepSeek tell), excessive boldface as "key takeaways", encyclopedic-yet-promotional drift, "moreover" overuse, "myths busted" contrast pattern, statistical regression to mean, subject puffery, level-2 heading skipping (Wikipedia-context-only), notability-language echoing (Wikipedia-context-only).
    - Wikipedia-specific signs are kept and labeled with explicit "Context: Wikipedia/encyclopedic prose" so reviewer applies them only when context fits.
    - Reviewer rigor is frequency-aware/judgment — single legitimate uses don't trigger HIGH; AI-typical frequency or formulaic positioning does.
    - Existing well-tested kill-list and patterns must survive the restructure intact.
- **Mode:** thorough
- **Interview:** thorough
- **Medium:** local

## 2. Approach

- **Architecture:**
    - Single-source restructure of `claude-plugins/writing/skills/human-writing/references/ai-tells-and-fingerprints.md` — reorganize into thematic sections (vocabulary, structural, rhetorical, tonal, formatting, statistical), fold Wikipedia patterns topically.
    - Top-tier patterns surface to `claude-plugins/writing/skills/human-writing/SKILL.md` (era-tracked vocab pointer, curly quotes, heading capitalization, "moreover", encyclopedic-promotional drift, excessive boldface). Deep detail stays in references (progressive disclosure).
    - `claude-plugins/writing/agents/writing-reviewer.md` extended to detect new patterns with frequency-aware judgment.
    - Plugin version bumped to v1.3.0 (minor — new patterns + new reviewer checks); CHANGELOG entry; READMEs synced where descriptions touch updated coverage.
    - Verbatim Wikipedia article NOT archived in repo; `/tmp/signs-of-ai-writing.md` consumed during /do, discarded.

- **Execution Order:**
    - D1 (restructured reference, deepest content) → D2 (SKILL.md surfacing) → D3 (reviewer extension) → D4 (versioning + READMEs + CHANGELOG)
    - Rationale: references file holds canonical content; SKILL.md cherry-picks from it; reviewer enforces it; versioning/docs follow once content is settled.

- **Risk Areas:**
    - [R-1] `/tmp/signs-of-ai-writing.md` missing or empty when /do starts | Detect: file existence + non-trivial size check before D1 begins
    - [R-2] Restructure silently weakens existing kill-list, anti-patterns, statistical-signature tables | Detect: content audit comparing pre/post-restructure for every existing pattern
    - [R-3] Reviewer additions cause false-positive flood on legitimate prose | Detect: dry-run reviewer on existing draft samples, look for HIGH flags on benign prose
    - [R-4] SKILL.md bloats past progressive-disclosure boundary when surfacing top-tier patterns | Detect: prompt-reviewer flags front-loading; approximate length guard

- **Trade-offs:**
    - [T-1] SKILL.md visibility vs progressive disclosure → Prefer top-tier patterns surfaced; deep era-tracked detail stays in references
    - [T-2] Reviewer strictness vs noise → Prefer frequency-aware judgment over flag-every-instance
    - [T-3] Wikipedia-specific signs included vs filtered → Prefer included with context labels (lossless; reviewer applies judgment)

## 4. Process Guidance

- [PG-1] Skill content optimizes for LLM consumption — scannable rules, do/don't tables, concrete examples — not human-readable prose explanation. The skill is loaded into LLM context, not read by humans.
- [PG-2] High-signal additions only. Every Wikipedia-derived pattern added must materially improve detection of AI tells in the writing plugin's target use cases (blog posts, articles, marketing copy, emails, social media). Drop low-signal patterns rather than pad.
- [PG-3] Reviewer applies frequency-aware judgment, not flag-every-instance. A single "moreover" or one curly quote in otherwise human prose is not HIGH; AI-typical density or formulaic positioning is. Apply same judgment to em-dashes (already in place).
- [PG-4] Wikipedia-specific signs (notability-language echoing, level-2 heading skipping, encyclopedic-tone-as-promotional-when-context-isn't-encyclopedic) are tagged with explicit "Context: Wikipedia/encyclopedic prose" labels in the reference file so the reviewer applies them only when context fits the writing target.
- [PG-5] No verbatim copy of the Wikipedia article is committed to the repo. `/tmp/signs-of-ai-writing.md` is consumed during /do and discarded.
- [PG-6] Use the prompt-engineering skill when authoring the SKILL.md, references, and reviewer additions. Apply first-principles prompt structure (WHAT and WHY, not HOW) consistent with the existing files.
- [PG-7] Calibrate emotional tone of additions to "trusted advisor" register — no urgency language, no excessive praise, no pressure framing. Match existing tone of the writing plugin's prompts.
- [PG-8] Prefer editing in chunks over a single large Write call when authoring or restructuring large markdown files. Catches drift early; reviewable diffs.
